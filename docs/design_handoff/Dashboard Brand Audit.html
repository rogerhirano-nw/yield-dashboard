Bringing the revenue dashboard onto the Newsweek system

A token-level review of the Streamlit ad-revenue dashboard against the Newsweek Design System — what reads off-brand today, and the exact CSS variables to fix it without disturbing the dashboard's severity logic.

Two constraints, resolved

Before any restyle: the calls that shape everything else

You flagged two hard constraints. Neither lets us drop the Newsweek system in wholesale — both need an explicit decision, made here rather than assumed.

Go light — Paper canvas

Re-skin to Newsweek Paper

Newsweek is a light, print-rooted brand: warm Paper, Ink text, serif headlines. The dashboard runs dark only because base="dark" is pinned. With that pin lifted, the on-brand move is the honest one — re-skin to Paper #FEFCF6.

The cost is contained: the green/amber/red severity grammar is preserved, only re-toned for a light canvas — darker state text on pale tints, so contrast holds at AA. A warm-ink dark ramp stays in the back pocket as a fallback if dark ever has to return.

The reds don't collide

Split brand red from state red

Newsweek Red is the identity; your dashboard already spends red on breach. Two reds, two jobs. We separate them by token namespace and — critically — by plane.

--brand-red (#E91D0C) is chrome only: the mark, the active-tab underline, the eyebrow tick. It never touches a number. --state-critical owns the data plane for breaches, as a distinct warm red used mostly as a tint, not a fill. Because chrome lives at the top and data in the body, the two reds never compete in one glance.

The translation

Eight moves that make a dark dashboard read as Newsweek

  1. 1Paper, not charcoalWarm Paper #FEFCF6 with Ink text — the native Newsweek canvas, not a dark SaaS shell.
  2. 2Serif for headers & figuresBenton Modern Display on the H1 and the big KPI numbers. The serif/sans contrast is the brand.
  3. 3Franklin Gothic for the UILabels, eyebrows, table text, metadata — one sans, replacing Streamlit's default.
  4. 4Eyebrows do the labellingUppercase, 0.08em tracked, semibold. Section and KPI labels become editorial kickers.
  5. 5Tabular numerals everywherefont-feature-settings "tnum" so figures align column-to-column.
  6. 6Severity as tint, not shoutGreen/amber/red as low-opacity surface + saturated text, banded in cells — not full-bleed fills.
  7. 7Rules over boxes1px warm hairlines and a heavier section rule replace heavy panel borders and shadows.
  8. 8Square corners4px on tiles and cells, 8px on banners, pills only for status and ordinals.
The fix, as tokens

Drop-in CSS variable block

Paste this into your Streamlit token block (e.g. a st.markdown <style> injection). Every recommendation below resolves to one of these variables — recolor or rescale the whole dashboard by editing this tier alone.

newsweek-dashboard.tokens.css 
/* ---- BRAND CHROME — identity only, never on data ---- */
--brand-red:        #e91d0c;   /* mark, tab underline, eyebrow tick */
--brand-red-strong: #c41608;   /* hover / pressed chrome */

/* ---- SURFACE — warm Paper (the native Newsweek canvas) ---- */
--surface-0:   #fefcf6;   /* app background — Newsweek Paper */
--surface-1:   #ffffff;   /* panels, tiles — lifted off paper */
--surface-2:   #f6f2e6;   /* sunk rows, hover fills */
--border:      #e7e0c9;   /* warm hairline */
--border-strong:#1f1e19;   /* 2px editorial ink rule */

/* ---- TEXT — Ink on paper ---- */
--text-primary:   #1f1e19; /* Newsweek Ink */
--text-secondary: #57564f;
--text-muted:     #8c887b;

/* ---- STATE — green / amber / red severity scale ---- */
--state-positive: #3c6b14;  --state-positive-surface: rgba(76,122,25,.12);
--state-warning:  #8a6d00;  --state-warning-surface:  rgba(214,170,0,.18);
--state-critical: #c41608;  --state-critical-surface: rgba(233,29,12,.10);

/* ---- DATA-VIZ — Newsweek secondary palette (toned for paper) ---- */
--viz-1: #4b62e0; --viz-2: #2d8d92; --viz-3: #824dd7;
--viz-4: #d84f86; --viz-5: #b08900; --viz-6: #5f9e2a;

/* ---- TYPE — serif editorial + sans UI ---- */
--font-sans:    "Franklin Gothic", "Helvetica Neue", Arial, sans-serif;
--font-display: "Benton Modern Display", Georgia, serif;
--type-eyebrow: 600 12px/1 var(--font-sans);   /* + uppercase, .08em */
--type-h1:      700 22px/1.15 var(--font-display);
--type-kpi:     700 30px/1.05 var(--font-display);
--type-body:    400 13px/1.5 var(--font-sans);
--num-feature:  "tnum" 1;   /* tabular numerals, all figures */
--track-eyebrow:0.08em;

/* ---- SPACING (4px base) & RADIUS (square-ish) ---- */
--space-1:4px; --space-2:8px; --space-3:12px; --space-4:16px; --space-6:24px;
--radius-sm:  4px;   /* tiles, inputs, banded cells */
--radius-md:  8px;   /* severity banners */
--radius-pill:999px; /* status + ordinal pills */
Findings

What's off-brand now, and the token that fixes it

Ordered by visual weight. Each fix is the variable above — nothing here asks you to touch component logic.

AreaOff-brand todayNewsweek-aligned fixToken
Surface Cold blue-charcoal (#0E1117 / #262730) reads generic-SaaS, not editorial. Warm Paper canvas #FEFCF6 with Ink text — the native editorial surface. --surface-0…2
UI type Streamlit's default Source Sans / system stack. Franklin Gothic for all labels, tables, metadata. --font-sans
Headers & figures Same sans, just bolded — no editorial contrast. Benton Modern Display serif on H1 + KPI numbers. --font-display
Eyebrows / labels Sentence-case grey labels, low contrast, no rhythm. Uppercase, 0.08em tracked, semibold kickers. --type-eyebrow
Accent red Streamlit #FF4B4B on buttons, links, charts and state — red means four things. Brand red = chrome only; severity owns its own red. --brand-red · --state-critical
Severity grammar Ad-hoc saturated green/amber/red fills, inconsistent across views. Tokenized scale: tint surface + saturated text + thin border. --state-*-surface
Data-viz palette Default Streamlit / Altair category colors. Newsweek secondary palette, lifted for dark legibility. --viz-1…6
Numerals Proportional figures — columns don't align. Tabular numerals on every metric and table cell. --num-feature
Radius Rounded 8px everywhere, including data tiles. 4px tiles/cells, 8px banners, pills for status only. --radius-sm…pill
Rules & dividers Faint or absent; structure carried by panel borders. 1px warm hairline + heavier section rule, editorial. --border · --border-strong

Rollout
  1. Unpin base="dark" in config.toml, paste the token block into the dashboard's <style> injection, and load the two webfonts.
  2. Re-point existing CSS to the variables — surfaces, text, severity, radius — starting with the Campaigns view.
  3. Swap the chart color sequence (Altair / line / bar) to --viz-1…6.
  4. Audit every red: if it's not the mark, a tab, or a breach, it shouldn't be red.
  5. Let the other six views inherit — they reuse these same tiles, banners, grids and pills.
